### Internet Research via MCP Servers

- Use the Search or Fetch MCP Server to search the internet when additional documentation or information is needed
- **CRITICAL**: Ensure searches do not contain internal, project-specific information
- Search queries must be generalized and focus on the relevant technology, API, or question without revealing concrete code snippets or internal structures
- Examples of appropriate searches:
  - "Spring Boot 3 testing best practices"
  - "JUnit 5 parameterized test examples"
  - "Clean Architecture domain factory methods"
- Examples of inappropriate searches:
  - Internal class names or project-specific terminology
  - Concrete implementation details
  - Project-specific architecture or domain models

## Allowed Research Context

You MAY search for:

- Generic Spring Boot, Maven, PostgreSQL documentation
- General TDD and Clean Architecture patterns
- Standard Java best practices
- General event-driven architecture patterns
- Generic microservice design principles

Always generalize your search queries to focus on the technology, not the specific project implementation.

**Note**: For project-specific search restrictions, see `.roo/rules/01-kagi-project.md`
